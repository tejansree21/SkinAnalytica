Introduction

What is SkinAnalytica?

SkinAnalytica is an ISIC-grade dermoscopy AI platform built for skin cancer researchers and clinical dermatologists. It combines a three-model deep learning ensemble with an 8-agent QA pipeline and a conversational research assistant — all accessible through a black-and-red web UI or REST APIs.

🧠
3-Model Ensemble
EfficientNetV2-S, ViT-Large, and ConvNeXt-Large combined via temperature-scaled weighted softmax.
🤖
8 AI Agents
Autonomous pipeline: self-healing → verification → active learning → drift → fairness → calibration.
💬
Research Assistant
Conversational AI on top of your data — answers questions about batches, models, and dermoscopy literature.
🏥
FHIR + ICD-10
Every scan exports a FHIR R4 DiagnosticReport with ICD-10 codes and NCCN treatment recommendations.
📊
Fairness Reports
Automated bias analysis across sex, age, and anatomical site. Fitzpatrick skin type via ISIC 2024 SLICE-3D.
🗺️
Grad-CAM++ + TIxAI
Per-image explanation maps with trustworthiness index scores showing model attention on lesion regions.
⚠ SkinAnalytica is a research tool. Results require clinician review before any diagnostic or treatment decision. Not a medical device.
Getting Started

Quickstart

Get SkinAnalytica running locally in three steps.

1

Start the APIs

bash
# Terminal 1 — Core API
cd SkinAnalytica/api
..\venv\Scripts\Activate.ps1
uvicorn SA05_api:app --host 0.0.0.0 --port 8001 --reload

# Terminal 2 — Research Assistant
cd SkinAnalytica/assistant
uvicorn assistant_api:app --port 8002

# Terminal 3 — Delivery
cd SkinAnalytica/delivery
uvicorn delivery_api:app --port 8003
2

Open the UI

bash
start skinanalytica_shell.html

The shell auto-detects API status and shows a green dot when :8001 is live.

3

Analyze an image

python
import requests

with open("lesion.jpg", "rb") as f:
    res  = requests.post("http://localhost:8001/analyze", files={"file": f})
    data = res.json()

print(data["verdict"])     # CANCER_FLAGGED / REVIEW_REQUIRED / NORMAL
print(data["confidence"])  # 0.882
print(data["icd10"])       # C43.9
💡 Set ANTHROPIC_API_KEY in your environment to enable full knowledge-grounded answers in the Research Assistant.

Models & Training

Three architectures trained on ISIC 2018 + 2019 + 2020 (68,472 images, 7 classes). Each trained independently, combined via weighted ensemble.

ModelParamsImg SizeVal AUCMel AUCEpochs
tf_efficientnetv2_s82M2240.97150.909830

convnext_large196M2240.98830.970627 (ES)

Training Strategy

Loss · EnhancedFocalLoss
γ=2.0, α=0.75, label smoothing=0.1. Handles 58:1 class imbalance (NV:DF) without collapsing to majority class.
Sampling · WeightedRandomSampler
Per-class inverse-frequency weights. Rare classes (DF, VASC) seen proportionally more often per epoch.
Augmentation · Mixup + CutMix
α=0.2 for both. Hair removal via black-hat morphology before all augmentation (dermoscopy-specific).
Optimizer · AdamW
Differential LR: backbone 5e-6, head 1e-4. Cosine annealing + warmup. Gradient clip 1.0. AMP enabled.

Ensemble weights

Weights are optimised on the validation set to maximise macro AUC. ViT-Large dominates (0.835) due to its superior individual performance.

ModelWeightRationale
efficientnetv2-s0.0511Strong calibration, lower raw AUC — minority contributor
vit-large-patch16-2240.8346Highest individual AUC — majority contributor
convnext-large0.1143Near-ViT performance, adds diversity in error patterns
⚠ Temperature T=0.4095 (<1) sharpens predictions. Label smoothing during training caused the ensemble to underestimate its own confidence — T scaling corrects this.
Platform

Explainability

SA03 generates per-image Grad-CAM++ heatmaps and TIxAI trustworthiness scores using all three ensemble models.

🗺️
Grad-CAM++
Second-order gradient weighting for precise lesion localisation. More accurate than standard Grad-CAM on small discriminative regions.
📐
TIxAI Score
Trustworthiness Index: measures how well model attention aligns with the actual lesion vs background. Score >0.7 = trustworthy.
🎲
MC Dropout
20-run Monte Carlo uncertainty estimate. High std = model is uncertain about this image region — useful for active learning.

Agents

Research Assistant

Conversational AI on top of your platform data. Routes questions to the data layer (live JSON results) or Claude API (dermatology knowledge). Maintains session history.

Data-grounded (Q01–Q10)
Queries your actual saved outputs — batch results, drift reports, ensemble metrics, annotation conflicts.
Knowledge-grounded (Q11–Q20)
Answers from published dermatology literature and ISIC standards. Requires ANTHROPIC_API_KEY for full responses.
python
from assistant.research_assistant import ResearchAssistant

ra  = ResearchAssistant(session_id="paul_meeting")
ans = ra.ask("What is our current MelAUC?")
print(ans["answer"])

# Or use pre-built questions
ans = ra.ask_prebuilt("Q06")  # Ensemble metrics
